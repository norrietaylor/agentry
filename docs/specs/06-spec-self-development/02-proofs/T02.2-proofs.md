## Summary

Created `tests/unit/test_git_diff_resolution.py` with 42 unit tests covering:

1. `_is_git_ref()` correctly identifies: HEAD, HEAD~1, HEAD~3, HEAD^2, HEAD^,
   SHA prefixes (7-40 hex chars), range syntax (main..feature, origin/main..HEAD),
   named refs (main, origin/main, feature/my-branch, v1.2.3).

2. `_is_git_ref()` correctly rejects: raw diff text (starts with "diff --git"),
   multi-line text, empty string, whitespace-only, arbitrary strings with spaces,
   strings with shell metacharacters.

3. `_resolve_git_diff()` runs git diff in a real temp git repo for valid refs
   and returns the diff output.

4. `_resolve_git_diff()` returns raw text directly when input is not a git ref
   (no git command is called).

5. `_resolve_git_diff()` falls back to raw text when git diff fails (invalid/
   nonexistent ref).

6. `_resolve_git_diff()` raises `NotAGitRepositoryError` for git refs when the
   target directory is not a git repository.

7. `_resolve_git_diff()` uses the `target` directory from the spec dict when
   provided, and falls back to `os.getcwd()` otherwise.

8. Integration through `resolve_inputs()`: `--input diff=HEAD~1` resolves
   correctly to the git diff output, and other cases (raw text, non-git dir,
   invalid ref fallback) behave as expected.
