Add a `pr:create` tool binding to `GitHubActionsBinder` that creates a branch,
commits changes, and opens a PR via the GitHub REST API.

## Implementation Summary

1. Added `"pr:create"` to `SUPPORTED_TOOLS` frozenset.
2. Implemented `_make_pr_create()` method returning a callable with the required signature.
3. The callable uses the GitHub REST API to:
   - Get the base branch SHA (`GET /repos/{owner}/{repo}/git/ref/heads/{base}`)
   - Create blobs for specified files (`POST /repos/{owner}/{repo}/git/blobs`)
   - Create a tree (`POST /repos/{owner}/{repo}/git/trees`)
   - Create a commit (`POST /repos/{owner}/{repo}/git/commits`)
   - Create a branch ref (`POST /repos/{owner}/{repo}/git/refs`)
   - Open a PR (`POST /repos/{owner}/{repo}/pulls`)
   - Add label (`POST /repos/{owner}/{repo}/issues/{number}/labels`)
4. Returns dict with `branch`, `pr_url`, and `status` fields.
5. Enforces safety guardrails: no force-push, no push to protected branches, no auto-merge.
6. Wired `pr:create` in `bind_tools()` method.
7. Handles errors with informative messages (403 permissions, 404 not found, 422 validation)
   following the pattern of `_make_pr_comment()` and `_make_pr_review()`.
