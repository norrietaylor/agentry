## Task
Implement `map_outputs()` on `GitHubActionsBinder` to write agent output to the filesystem and post PR comments.

## Implementation

**File modified:** `src/agentry/binders/github_actions.py`

- Replaced the `NotImplementedError` stub in `map_outputs()` with a full implementation
- Added `_post_output_comment()` private helper for GitHub API interaction

**Key behaviors:**
1. Always writes output to `$GITHUB_WORKSPACE/.agentry/runs/<run_id>/output.json` (creates directories as needed)
2. Always includes `execution_record` path in the return mapping
3. On PR events: reads `output.json` content (or falls back to a descriptive message) and posts it as a PR comment via `POST /repos/{owner}/{repo}/issues/{number}/comments`
4. On non-PR events: skips comment posting entirely
5. Extra `output_paths` from declarations are included in the mapping
6. API errors produce structured `RuntimeError` messages: 403 scope error, 404 not found, network timeout
