## Implementation

Added `resolve_inputs()` method to `GitHubActionsBinder` in
`src/agentry/binders/github_actions.py`, replacing the `NotImplementedError`
stub from T01.1 with a full implementation.

### Key behaviors implemented

1. **repository-ref**: Resolves directly to `self._workspace`
   (`GITHUB_WORKSPACE` environment variable).

2. **git-diff**: Fetches PR unified diff via GitHub API using httpx:
   `GET /repos/{owner}/{repo}/pulls/{number}` with
   `Accept: application/vnd.github.diff`. PR number extracted from event
   payload parsed during construction. Raises `ValueError` with actionable
   message when event is not `pull_request`.

3. **string** (multi-source resolution in priority order):
   - `provided_values[name]` — explicit `--input` CLI override
   - `event_payload["inputs"][name]` — `workflow_dispatch` event inputs
   - Dot-notation `source` key in input spec (e.g., `"issue.title"`)
   - Returns `None` for optional inputs with no resolvable source
   - Raises `ValueError` for required inputs with no resolvable source

### Private helpers added

- `_resolve_git_diff(input_name)` — GitHub API call via httpx
- `_resolve_string(name, spec, provided_values)` — multi-source strategy
- `_traverse_payload(dotpath)` — dot-notation event payload traversal
