1. Added `--binder NAME` option to `agentry run` command in `src/agentry/cli.py`

   - Help: "Override binder selection. Auto-detected when not specified: github-actions if GITHUB_ACTIONS=true, local otherwise."

2. Auto-detection logic added:

   - If true: uses "github-actions" binder
   - Otherwise: uses "local" binder

3. Registered `github-actions` as a built-in binder in `src/agentry/binders/registry.py`
   - Added `GITHUB_ACTIONS_BINDER_NAME = "github-actions"` constant
   - Added `GitHubActionsBinder` to the built-in `discover_binders()` map
   - Added fast-path for `get_binder("github-actions")`

4. Used `get_binder(name)` from `agentry.binders.registry` to instantiate binder
   - Added error handling for `KeyError` (unknown binder name)
   - Added error handling for `ValueError` (binder init failure)

5. Wired `GitHubTokenScopeCheck` into preflight checks when binder is "github-actions":
   - Extracts `tool_declarations` from `_loaded_workflow.tools.capabilities`
   - Gets `GITHUB_REPOSITORY` from environment
   - Creates `GitHubTokenScopeCheck` instance and appends to checks list

6. Replaced hardcoded `LocalBinder()` in composition engine path with `_active_binder`
   - Removed `from agentry.binders.local import LocalBinder` and `_binder = LocalBinder()`
   - Passes `_active_binder` to `CompositionEngine(binder=_active_binder, ...)`

7. `--binder local` works as explicit override even when `GITHUB_ACTIONS=true`
