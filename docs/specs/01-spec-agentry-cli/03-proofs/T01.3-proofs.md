## Task

Implement the YAML loading layer using `yaml.safe_load()`, feed parsed data into
Pydantic v2 models, implement structured error reporting (file path, field path,
remediation hints), create the `validate` CLI command, and create test fixtures.

- `tests/unit/test_cli.py` - Updated existing CLI test to use valid workflow YAML

## Key Behaviors Verified

1. `validate_workflow_file()` returns `[]` for valid workflow YAML (exit 0)
2. `validate_workflow_file()` returns structured error list for invalid YAML (exit 1)
3. Error format: `error[type]: <file>: <field.path>` with `hint:` remediation
4. Unknown keys report the full field path (e.g. `model.extra_field`)
5. Semver errors report `identity.version` with hint
6. Missing discriminator field (`type`) is reported
7. Unresolved `$variable` references are reported
8. CLI `agentry validate` exits 0 on success, 1 on failure
9. JSON output mode: `{"status": "valid", "path": "..."}` on success
10. Errors printed to stderr; success to stdout
