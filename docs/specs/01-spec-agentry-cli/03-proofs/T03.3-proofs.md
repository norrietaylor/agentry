## Implementation Summary

Implemented concrete tool bindings in `src/agentry/binders/local.py`:

### `repository:read`
- Factory function `_make_repository_read()` returns a callable with signature
  `repository_read(*, repo_root: str, path: str) -> str`
- Uses `Path.resolve()` to follow symlinks before checking containment
- Rejects paths that resolve outside `repo_root` via `PathTraversalError`
- Raises `FileNotFoundError` for missing files, `IsADirectoryError` for directories

### `shell:execute`
- Factory function `_make_shell_execute()` returns a callable with signature
  `shell_execute(*, command: str, cwd: str | None = None) -> str`
- `_validate_shell_command()` parses commands with `shlex.split()` and validates:
  - Executable must be in `_SHELL_ALLOWLIST`: git, ls, find, grep, cat, head, tail, wc
  - For `git`, sub-command must be in `_GIT_SUBCOMMAND_ALLOWLIST`: log, diff, show, blame
  - Absolute paths to executables (e.g. `/usr/bin/cat`) are handled via `Path.name`
- Raises `CommandNotAllowedError` for any disallowed command or git sub-command

### New Exceptions
- `PathTraversalError` in `src/agentry/binders/exceptions.py`
- `CommandNotAllowedError` in `src/agentry/binders/exceptions.py`
- Both exported via `src/agentry/binders/__init__.py`

## Security Edge Cases Covered

- `../` path traversal in `repository:read`
- Symlinks pointing outside repo root
- Absolute paths outside repo root
- Blocked commands: rm, echo, curl, wget, python3, bash, chmod, mv, cp, touch, mkdir
- Blocked git sub-commands: commit, push, checkout, reset, rm, add, fetch, pull, rebase, merge
- Absolute executable paths (e.g. `/usr/bin/rm`)
- Empty command string
- `git` without sub-command
