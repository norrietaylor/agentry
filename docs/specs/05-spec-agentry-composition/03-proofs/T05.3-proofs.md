### Implementation Details

**task-decompose.yaml**:
- Accepts `triage_result` (string) and `repository-ref` inputs
- Uses `task-decompose-system-prompt.md` as system prompt
- Outputs structured task array with title, description, priority, and estimated_effort fields
- Timeout: 180 seconds, max_tokens: 4096

**task-decompose-system-prompt.md**:
- Instructs agent to break down triaged issues into 3-7 focused implementation tasks
- Defines priority levels (critical, high, medium, low)
- Defines effort estimates (small, medium, large, xl)
- Provides guidelines for security, performance, and feature issues

**planning-pipeline.yaml**:
- Composition workflow with three steps:
  1. `triage` step (depends on nothing) - runs triage.yaml
  2. `task-decompose` step (depends on triage) - takes triage output as input
  3. `summary` step (depends on task-decompose) - runs triage.yaml
- Demonstrates composition pattern with step chaining and data passing
