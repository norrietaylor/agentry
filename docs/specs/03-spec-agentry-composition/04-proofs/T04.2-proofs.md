# T04.2 Proof Summary: Integrate data passing into CompositionEngine

## Task

Modify `src/agentry/composition/engine.py` to integrate file-based data passing
into the node execution flow.

## Changes Made

1. **Replaced `_resolve_node_inputs()` placeholder** with a call to
   `data_passing.resolve_node_inputs()`, passing `_node_outputs` and
   `_node_failures` dicts.

2. **Added `_node_outputs: dict[str, Path]`** to track result.json paths for
   successfully completed nodes. Populated in `_execute_node` after success and
   in the retry-success branch.

3. **Added `_node_failures: dict[str, Path]`** to track result.json paths for
   nodes that failed under the skip policy. Populated in the `skip` and
   `retry->skip fallback` branches of `_apply_failure_policy`.

4. **Resolved paths are injected as `resolved_inputs`** when calling
   `runner.provision()` and `AgentConfig`, making upstream node outputs
   available to downstream nodes.

5. **Upstream completion verification** is handled by `data_passing.resolve_node_inputs()`
   which raises `DataPassingError` if a referenced node is not in either
   `_node_outputs` or `_node_failures`.
