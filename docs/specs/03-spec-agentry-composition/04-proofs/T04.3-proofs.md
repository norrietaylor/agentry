## Test Coverage

### Unit Tests (`tests/unit/test_data_passing.py`)

- **TestResolveFullOutputReference** (5 tests): Verifies `A.output` resolves to the
  absolute path of A's `result.json`. Covers single input, string return type, file
  existence, empty inputs, and multiple inputs.

- **TestResolveFieldExtraction** (3 tests): Verifies `A.output.severity` extracts the
  named field and writes it to a temp file. Covers string and integer fields, and
  missing field error.

- **TestResolveFailurePropagation** (3 tests): Verifies that when A fails with skip
  policy, B's input referencing `A.output` resolves to A's NodeFailure JSON path
  (containing `_failure: true` sentinel). Also verifies failure path takes priority
  over output path.

- **TestResolveMissingUpstream** (3 tests): Verifies `DataPassingError` is raised when
  referenced upstream node has no output and is not in failures map.

- **TestWriteNodeOutput** (6 tests): Verifies `write_node_output` creates node directory,
  writes `result.json` with correct content, returns absolute path, and creates
  intermediate directories.

- **TestExtractField** (9 tests): Verifies `extract_field` reads string/integer/list/dict
  fields, raises for missing field, missing file, invalid JSON, non-object JSON, and
  includes field name in error message.

### Integration Tests (`tests/integration/test_composition_pipeline.py`)

- **TestThreeNodePipeline** (11 tests): Full A->B->C pipeline where each node receives
  the previous node's output as input. Verifies:
  - All three nodes complete with `NodeStatus.COMPLETED`
  - Overall `CompositionStatus.COMPLETED`
  - Each node's `result.json` exists in `<run_dir>/<node_id>/result.json`
  - B received A's `result.json` path as `prev_output` input
  - C received B's `result.json` path as `prev_output` input
  - A has no resolved inputs (no upstream nodes)
  - `composition-record.json` saved to disk with all node statuses
