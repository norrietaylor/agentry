# T05.2 Proof Summary: Add composition progress display with TTY-aware formatting

## Task

T05.2: Add composition progress display with TTY-aware formatting

## Implementation

Created `src/agentry/composition/display.py` with `CompositionDisplay` class:

1. **Node start event**: `on_node_start(node_id)` — emits `[*] Running node 'X'...` (non-TTY) or starts a background spinner (TTY).
2. **Node complete event**: `on_node_complete(node_id, duration)` — emits `[OK] Node 'X' completed (2.3s)`.
3. **Node fail event**: `on_node_fail(node_id, error)` — emits `[FAIL] Node 'X' failed: <error>`.
4. **Node skip event**: `on_node_skip(node_id)` — emits `[SKIP] Node 'X' skipped (upstream failure)`.
5. **Execution summary**: `print_summary(record)` — emits overall status, per-node status table, total wall-clock time.
6. **JSON output mode**: When `output_format="json"`, all event and summary output is suppressed (caller emits full CompositionRecord as JSON).
7. **TTY detection**: Spinner (`_Spinner`) runs in a background thread in TTY mode; plain lines in non-TTY mode.

Modified `src/agentry/composition/engine.py`:
- Added `on_node_start`, `on_node_complete`, `on_node_fail`, `on_node_skip` optional callback parameters to `CompositionEngine.__init__`.
- Callbacks are fired at the appropriate lifecycle points in `_execute_node`.
- Per-node start timestamps tracked in `self._node_start_times` for duration reporting.

Modified `src/agentry/cli.py`:
- Imports `CompositionDisplay` from `agentry.composition.display`.
- Creates `_display = CompositionDisplay(is_tty=_is_tty_mode, output_format=output_format)`.
- Wires display callbacks into `CompositionEngine` constructor.
- Calls `_display.print_summary(_composition_record)` after execution (text mode) or emits full JSON record (JSON mode).

Updated `src/agentry/composition/__init__.py` to export `CompositionDisplay`.

## Key Design Decisions

- `CompositionDisplay` is a standalone module (`display.py`) rather than inline in `cli.py` for testability and separation of concerns.
- The `_Spinner` runs in a daemon thread so it doesn't block shutdown; `stop()` joins with a 1-second timeout.
- Callbacks are optional (defaulting to `None`) so `CompositionEngine` can be used without display (e.g. in tests or programmatic use).
- JSON mode suppresses all display output in `CompositionDisplay`; the caller (CLI) is responsible for emitting the `CompositionRecord.to_dict()` JSON.
- `_is_tty` is read from the Click context object (populated by the root `main` command) to ensure consistent TTY detection.
