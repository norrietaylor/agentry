# T05.1 Proof Summary: Add composition detection and --node flag to agentry run

## Task
T05.1: Add composition detection and --node flag to agentry run

## Implementation
Modified `src/agentry/cli.py` to extend the `run` command with:
1. `--node NODE_ID` option for single-node isolation execution
2. Composition detection: checks `workflow.composition.steps` for non-empty list
3. Composition dispatch: routes through `CompositionEngine` with `RunnerDetector` and `LocalBinder`
4. Single-agent fallback: non-composition workflows unchanged
5. Error handling: `--node` on non-composition workflows emits error + exit 1

## Key Design Decisions

- `--node` flag implements isolation by creating a single-step `CompositionBlock` with
  no `depends_on` and no `inputs`, preventing upstream data passing and downstream propagation.
- `workflow_base_dir` is derived from the workflow YAML's parent directory so relative paths
  in `composition.steps[*].workflow` resolve correctly.
- CompositionEngine errors surface as exit code 1 with a descriptive message.
- If `CompositionEngine` is not yet importable, a graceful stub response is emitted.
