## Test Coverage

1. **test_composition_detection_calls_engine** — verifies `CompositionEngine.execute()` is called when `composition.steps` is non-empty.
2. **test_single_agent_fallback_uses_executor** — verifies `CompositionEngine` is NOT instantiated for a workflow with empty `composition.steps`.
3. **test_node_flag_with_composition_isolates_node** — verifies `--node triage` causes an isolated single-step composition block to be passed to the engine.
4. **test_node_flag_without_composition_emits_error** — verifies `--node X` on a non-composition workflow exits non-zero with a meaningful error message.
5. **test_json_output_format_contains_composition_record_fields** — verifies `--output-format json` emits valid JSON containing `overall_status`, `node_statuses`, and `wall_clock_timing` fields.
6. **test_text_output_format_contains_per_node_status** — verifies `--output-format text` emits human-readable output with per-node names and status.

## Implementation Notes

- `CompositionEngine` is imported locally inside the `run` command function body, so the correct patch target is `agentry.composition.engine.CompositionEngine`.
- `AsyncMock` is used for `engine.execute()` since the CLI calls it via `asyncio.run()`.
- The canned `CompositionRecord` has two nodes (`triage`, `summary`) both with `NodeStatus.COMPLETED` and `CompositionStatus.COMPLETED`.
