# T05.3 Proof Summary: Signature Verification During Setup Phase

## Task
Implement signature verification in `src/agentry/security/signing.py` and integrate it with `SetupPhase` in `src/agentry/security/setup.py`. Write tests for the verify cycle.

## Implementation Summary

### Files Modified
- `src/agentry/security/setup.py`: Added `SetupSignatureError` exception, `workflow_path` and `public_key_path` constructor parameters, and `_verify_signature()` private helper method called as Step 5 in `SetupPhase.run()`.

### Files Created
- `tests/unit/test_signing_verification.py`: 12 unit tests covering the full verification cycle.

### Key Behaviour
1. **Opt-in verification**: Verification is skipped when no `workflow_path` is provided to `SetupPhase`.
2. **Signature present + valid**: `SetupPhase.run()` proceeds normally when signature verifies.
3. **Signature present + invalid**: `SetupSignatureError` is raised with the message from `SignatureVerificationError`, including the signed timestamp.
4. **No signature block**: Verification is skipped silently.
5. **Public key absent**: Verification is skipped silently (key must be explicitly deployed to enforce signing).
