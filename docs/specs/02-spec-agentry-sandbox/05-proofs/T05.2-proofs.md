# T05.2 Proof Summary: Workflow Signing and 'agentry sign' CLI Command

## Task

Implement signing logic in `src/agentry/security/signing.py`. Sign the `safety` block and
`output.side_effects` block of a workflow definition using Ed25519.  Serialize signed
blocks deterministically (`yaml.dump` with `default_flow_style=False`, `sort_keys=True`)
before signing.  Append a signature block to the workflow YAML.  Implement
`agentry sign WORKFLOW_PATH` CLI command.  Write tests for sign cycle and deterministic
serialization.

### Signature block format

```yaml
  algorithm: ed25519
  signed_blocks:
    - safety
    - output.side_effects

```

## Key Behaviors Verified

1. `sign_workflow()` appends a `signature` block with algorithm, signed_blocks, signature (hex), timestamp.
2. Only `safety` and `output.side_effects` are signed.
3. Ed25519 signatures are 64 bytes (128 hex chars).
4. Timestamps are ISO 8601 UTC (YYYY-MM-DDTHH:MM:SSZ).
5. Deterministic serialization: same workflow + same key = identical signature.
6. `verify_workflow_signature()` passes for unmodified workflows.
7. `verify_workflow_signature()` raises `SignatureVerificationError` when safety block modified.
8. Modifying non-signed blocks (e.g., `identity`) does not invalidate the signature.
9. `agentry sign WORKFLOW_PATH` CLI exits 0 on success.
10. `agentry sign` exits 1 when workflow or key file is missing.
