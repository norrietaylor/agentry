## Requirements Implemented

1. **DNS query logging to execution record**: DNS queries (both resolved and blocked) are logged to `.agentry/runs/TIMESTAMP/execution-record.json` with a `dns_queries` section. Each entry includes `domain`, `action` (`resolved` or `blocked`), `query_type`, and `timestamp`.

2. **Network isolation verification**: `SetupPhase._verify_network_isolation()` now uses `NetworkIsolationVerifier` when a `dns_proxy` is present in runner metadata. It verifies that:
   - A known-blocked domain (`example.com`) returns `action=blocked`
   - The LLM API domain (`api.anthropic.com`) returns `action=resolved`
   - If verification fails, `NetworkIsolationError` is raised to abort setup

3. **Integration tests with `@pytest.mark.docker`**: `tests/integration/test_network_isolation.py` demonstrates that a container (via the DNS proxy logic) can resolve `api.anthropic.com` but cannot resolve `example.com` when it's not in the allowlist.
