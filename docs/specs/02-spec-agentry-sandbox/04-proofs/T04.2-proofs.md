## Requirements Implemented

- `AnthropicAPIKeyCheck`: verifies `ANTHROPIC_API_KEY` env var is set and non-empty, then makes a lightweight `GET /v1/models` call to confirm the key is accepted. Returns descriptive failure messages for missing key, 401 Unauthorized, other HTTP errors, and network errors.
- `DockerAvailableCheck`: skips (passes trivially) when trust level is not `"sandboxed"`. For sandboxed trust, checks docker executable is on PATH then runs `docker info` to verify the daemon responds. Handles timeout and exec errors.
- `FilesystemMountsCheck`: verifies all paths in `filesystem.read` and `filesystem.write` exist on the host. Reports all missing paths (not just the first), with count and path names in the failure message.
- All three checks satisfy the `PreflightCheck` protocol (`name` property + `run()` method returning a result with `passed`, `name`, `message`, `remediation` fields).
- All three checks integrate correctly with `PreflightChecker.run_all()`.
