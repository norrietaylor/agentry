## Design Decisions

- `AgentConfig` bundles agent execution parameters (system prompt, inputs, tool names, LLM config, retry config, timeout) to keep the `execute()` signature clean and extensible.
- `llm_config` and `retry_config` fields on `AgentConfig` use `Any` type to avoid a circular import between `agentry.runners.protocol` and `agentry.llm.models`.
- `ExecutionResult` wraps `ExecutionRecord` to layer container-specific fields (exit code, stdout, stderr, timed_out) on top of the existing agent execution record.
- `RunnerContext.metadata` is an open dict so DockerRunner can store image digest, resource limits applied, and other diagnostics for the setup manifest without changing the protocol.
