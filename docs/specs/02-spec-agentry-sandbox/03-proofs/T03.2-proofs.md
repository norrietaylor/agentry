# T03.2 Proof Summary: SetupPhase and setup manifest generation

## Task

Implement `SetupPhase` class in `src/agentry/security/setup.py` that:
1. Provisions execution environment via `runner.provision()`
2. Verifies network isolation
3. Runs preflight checks
4. Compiles the output validator schema
5. Generates a setup manifest JSON containing all required fields
6. Saves the manifest at `.agentry/runs/<TIMESTAMP>/setup-manifest.json`

## Key Design Decisions

- `fingerprint_credential` uses `hashlib.sha256` (stdlib, no external dep)
- Schema compilation uses `jsonschema.Draft7Validator.check_schema` when available; gracefully skips when not installed
- Manifest directory timestamp is filesystem-safe (no colons/dashes/plus signs)
- `SetupPhase` accepts an injected runner (RunnerDetector deferred to T01.6)
- Network isolation verification is logging-only; actual enforcement is at the Docker network layer
